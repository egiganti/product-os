## Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Frontend | Next.js 15 + TypeScript | SSR, App Router, type safety |
| Styling | Tailwind CSS + shadcn/ui | Speed + consistency |
| UI Components | Google Stitch exports | AI-generated, project-specific |
| Backend | Next.js API Routes | Colocation, simplicity |
| Database | Supabase (Postgres) | Auth + DB + Storage in one |
| Auth | Supabase Auth | Handles OAuth, magic links |
| Deploy | Vercel | Zero-config, edge network |
| CI/CD | GitHub Actions | product-os native |
| Monitoring | Sentry + Vercel Analytics | Errors + performance |
| Testing | Vitest + Playwright | Unit + E2E |

---

## Architecture

```
[Client]
   ↓ Next.js App Router (RSC + Client Components)
[API Layer]
   ↓ /api/* routes (Next.js) or external service
[Data Layer]
   ↓ Supabase client (server-side only for sensitive ops)
[Database]
   PostgreSQL (Supabase managed)
```

## Security Baseline

- Auth: Supabase RLS on all tables
- API: All routes validate session server-side
- Secrets: Environment variables only, never in code
- CORS: Restricted to production domain
- Headers: security headers via next.config.js
